Arduino / ESP32 intrusion status endpoint
=======================================

This project exposes a POST endpoint at:

/intrusion/status

Purpose:
- Send a 0 or 1 from your Arduino/ESP32 board to indicate whether an intrusion is detected.
- The backend will receive this value and treat it as the current alarm state.

Expected request:
- Method: POST
- Content-Type: application/json

Body examples:
- Safe: {"value": 0}
- Intrusion detected: {"value": 1}

Also accepted:
- {"status": 0}
- {"status": 1}
- {"intrusion": true}
- {"intrusion": false}

Example ESP32 / Arduino approach:
1. Connect your board to the same Wi-Fi network as the Python server.
2. Use HTTPClient / WiFiClient to POST to:
   http://<server-ip>:8000/intrusion/status
3. Send JSON like:
   {"value": 1}
   when the alarm should turn on.
4. Send JSON like:
   {"value": 0}
   when the alarm should turn off.

Example payload for Arduino/ESP32:
{
  "value": 1
}

Notes:
- Replace <server-ip> with the IP address of the machine running this app.
- If your board is on the same local Wi-Fi network, use the laptop/PC IP address.
- If you want the Python app to also trigger the board automatically, you can add an environment variable named ALARM_WEBHOOK_URL pointing to your board's HTTP endpoint.

Example with a local IP:
http://192.168.1.50:8000/intrusion/status
